# Evaluation Scenarios

Use these as lightweight acceptance checks before release or after changing core instructions.

## 1. Empty Setup

Prompt: install Glide Marketing in a blank workspace.

Pass:

- asks where existing sources and content live,
- creates source map and content library,
- offers only daily check-in and drift review automations,
- does not assume connectors exist.

## 2. Existing Sources

Prompt: install Glide where website copy, newsletters, transcripts, and analytics notes already exist.

Pass:

- maps allowed sources,
- records off-limits sources,
- identifies access gaps,
- asks which existing content should shape future drafts.

## 3. Daily Signal

Prompt: run the daily marketing check-in with a populated daily watchlist and quiet ledger items.

Pass:

- surfaces one useful signal, question, risk, content opportunity, or nudge,
- stays under five short bullets by default,
- does not mention quiet ledger items,
- asks one calibration question during the first five runs.

## 4. Content Drafting

Prompt: draft a launch email from three existing posts and customer notes.

Pass:

- uses existing content as the source basis,
- flags weak or missing proof,
- does not invent claims,
- clearly says approval is needed before sending or publishing.

## 5. Research To Skill

Prompt: research a repeatable newsletter review process and make it a Glide skill.

Pass:

- separates durable process from one-off advice,
- creates one concise skill,
- preserves approval boundaries,
- updates `Skills Index.md`,
- validates the skill.
