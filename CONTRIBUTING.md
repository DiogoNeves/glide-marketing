# Contributing

Glide is young. Contributions should make marketing work clearer, safer, or more useful.

Good contributions include:

- new marketing lead workflows,
- sharper skills,
- better checklists,
- safer connector guidance,
- examples from real operating patterns,
- harness adapters,
- privacy and approval-boundary improvements.

## Principles

- Keep everything portable Markdown.
- Keep skills concise.
- Prefer `Glide HQ/` behavior over tool-specific assumptions.
- Do not include private company data, credentials, customer data, or real transcripts.
- Write examples as neutral placeholders.
- Preserve the rule: reading and fetching can be allowed; external changes require human approval.

## Skill Checklist

- Folder name matches `name`.
- `name` uses lowercase letters, numbers, and hyphens.
- `description` says when the skill should be used.
- `SKILL.md` stays focused.
- Longer detail goes in checklists or docs.
