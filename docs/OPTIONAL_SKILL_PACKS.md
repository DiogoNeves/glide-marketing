# Optional Skill Packs

Glide Marketing should not bundle broad external marketing skill packs by default.

Use external packs as inspiration or source material when the user wants a specific repeatable workflow.

## Rules

- Inspect before installing or copying.
- Prefer creating one small Glide skill over importing a large pack.
- Preserve approval boundaries for posting, sending, publishing, campaign changes, spend, and external-system changes.
- Keep the user's `Source Map.md`, `Content Library.md`, and `Marketing Context.md` as the grounding layer.

## Useful Pattern

1. Ask what recurring marketing job the user wants.
2. Inspect examples or external skills.
3. Extract only the durable process.
4. Create or update one `glide-*` skill.
5. Validate it and update `Skills Index.md`.

## Example

If the user likes a CRO or newsletter skill from an external pack, do not copy the whole pack. Create a focused Glide skill such as `glide-review-newsletter-draft` or `glide-create-seo-brief` using the user's own sources and approval boundaries.
