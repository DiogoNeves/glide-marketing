# Release Privacy Checklist

Before publishing a release:

- Search for real company names, private customer data, personal data, credentials, local-only paths, and internal metrics.
- Verify examples use placeholders.
- Confirm `Glide HQ/` templates are generic.
- Confirm skills do not assume a specific company, industry, connector, or model provider.
- Confirm approval boundaries are intact.
- Rebuild history before first public release if private material ever entered git history.
