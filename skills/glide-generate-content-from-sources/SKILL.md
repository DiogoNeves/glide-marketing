# Glide Generate Content From Sources

## Load

- `Glide HQ/AGENTS.md`
- `Glide HQ/Source Map.md`
- `Glide HQ/Content Library.md`
- `Glide HQ/Company Context.md`
- `Glide HQ/Marketing Context.md`
- `Glide HQ/Marketing Lead Brief.md`
- Relevant source material when access is configured

## Process

1. Clarify the asset type, audience, goal, channel, and approval needs.
2. Use `Content Library.md` to find the best source set and voice reference.
3. Fetch or read source material only from allowed sources.
4. Extract reusable ideas, examples, claims, objections, phrases, and proof.
5. Draft the asset from existing material; do not invent proof.
6. Flag missing evidence, weak claims, sensitive material, or required approvals.
7. Offer a short revision path when useful.
8. Update `Content Library.md`, `Marketing Context.md`, or `Follow-Through Ledger.md` only when durable signal appears.

## Guardrails

- Drafting is allowed.
- Posting, sending, publishing, scheduling, or changing campaigns requires explicit approval.
- Do not copy private source text into durable files unless the user asks.
- If source access is missing, ask for the smallest useful input or add an access gap.

## Output

- Draft or outline.
- Source basis.
- Missing evidence.
- Approval needed before external use.
- Suggested Glide HQ updates, if any.
