# Nightly Marketing Research Review

Purpose: run a quiet 4am background pass that improves marketing memory, source coverage, research, content connections, and operating quality without turning the result into a daily report.

## Load

- `Glide HQ/AGENTS.md`
- `Glide HQ/Operating Manual.md`
- `Glide HQ/Company Context.md`
- `Glide HQ/Marketing Context.md`
- `Glide HQ/Marketing Lead Brief.md`
- `Glide HQ/Source Map.md`
- `Glide HQ/Content Library.md`
- `Glide HQ/Daily Watchlist.md`
- `Glide HQ/Connector Inventory.md`
- `Glide HQ/Open Loops.md`
- `Glide HQ/Questions Queue.md`
- `Glide HQ/Research/Research Index.md`
- `Glide HQ/Follow-Through Ledger.md`
- `Glide HQ/Contradiction Register.md`
- `Glide HQ/Decision Log.md`
- `Glide HQ/Evals/*.md` when present
- `Glide HQ/Evals/Nightly Research Audit.md`
- Recent user-agent exchanges when available
- Approved read-only connectors or app sources when useful

## Internal Plan

Start each run by deciding:

- which open loops or research questions have the highest marketing value;
- which source, content, or context gaps would improve future daily updates;
- which market, customer, channel, or content signals may be connected;
- which eval signals suggest process improvement;
- which checks can run in parallel.

Keep this planning internal unless approval is needed.

## Parallel Tracks

When subagents are available, run narrow tracks in parallel and integrate the results yourself.

Useful tracks:

- recent exchanges and missing marketing memory;
- open loops that require research;
- active research that needs deeper sources;
- source map and content library gaps;
- content reuse or distribution opportunities from existing material;
- evals and marketing-process improvements;
- stale working-memory candidates.

## Integration

After parallel work:

- update `Research Index.md`, `Source Map.md`, `Content Library.md`, or relevant context files when findings are durable;
- connect related source material, content ideas, questions, decisions, and open loops;
- refine queued questions instead of accumulating vague ones;
- consolidate duplicated loops or research entries;
- suggest a new marketing utility skill only when a repeatable process is emerging;
- prefer shorter broader rules over narrow case-by-case conditions.

## Stale Working Memory

Look for active working items unused for over a month.

Clear or retire an item only when it is plainly no longer relevant, duplicated, superseded, or too vague to help future marketing advice.

Do not delete ordinary workspace notes, source material, or valuable company memory. Preserve durable learning in the appropriate long-term file before removing an item from an active queue.

## Audit

Update `Glide HQ/Evals/Nightly Research Audit.md` with run date, files changed, reasons, queued research or memory gaps, and approval-needed items.

Keep only the last 4 days of audit detail. Older detail can be summarized or removed when it no longer helps future runs.

## Output

Do not produce a user-facing summary by default. Surface only approval requests, blockers, or discoveries that should not wait for the daily check-in.
