# Daily Watchlist

What the user wants Glide Marketing to check during daily updates.

This file should be created during installation and refined during the first five daily check-ins.

## Rules

- Track only checks the user actually wants.
- For each check, record what source or connector is needed.
- Mark access gaps clearly.
- Prefer read/fetch access; external changes require approval.
- Link important unresolved items to `Follow-Through Ledger.md`.
- During the first five daily check-ins, ask for more direction and refine this file from the answers.
- For each check, define what the agent should do when signal appears.
