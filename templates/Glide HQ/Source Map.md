# Source Map

Where Glide can look for marketing context and signal.

## Rules

- The user defines the sources.
- Prefer read/fetch access.
- Record access gaps clearly.
- Do not copy raw sensitive data here unless the user asks.
- Ask before changing any source system.

## Setup Questions

- Which sources should Glide read first?
- Which sources are useful but not connected yet?
- Which sources are sensitive or off-limits?
- Which daily signals depend on each source?
- What should Glide do when a source produces signal: mention, draft, add to ledger, ask, or ignore?
